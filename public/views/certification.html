Certify Extension

In order to begin the process to certify an extension you will need to select a github release to certify and what versions of Qlik Sense to certify against. Only a github release can be certified. Once you choose a release and which versions to certify against, the source code of the release will be automatically compared against the Extension Source Validation Guidelines. You can view these guidelines below. If the Extension Source Validation passes, the extension will be automatically and manually compared against the Extension Content Guidelines. You can view these guidelines below. The extension will also be tested to ensure it operates as expected.

Extension Source Validation Guidelines

In order for your extension to be certified, the release will be tested to ensure the source code contains the following:

  • A README file
  • A LICENSE file
  • A .qext file. This file needs to have the following fields:
    • name
    • description
    • author
    • author contact information
  • A .js file whose name matches the name of the .qext file
  • A "sample data" folder with sample data that can be used to test the extension

Extension Content Guidelines

When you submit below to certify the extension, you agree that all files in the release adhere to the following guidelines:

  • The README file contains the following:
    • A clear indication of the extension with a description, relevant screenshots
    • A detail of any extensions or technology that is used by the extension that requires licensing
    • Steps on how to set up the extension with the sample data provided
    • A clear indication of the responsiveness of the extension. This does not mean the extension needs to be responsive, but an indication of how responsive the extension is should be given
  • There are no hidden or undocumented features. All features of the extension are fully and correctly documented
  • The extension is well tested for performance. If the extension is expected to not perform quickly and efficiently, this should be documented in the README file
  • The code for the extension is well written and efficient
  • The files in the repository and any files downloaded for the extension do not contain objectionable content (pornography, illegal content, objectionable language, etc)
